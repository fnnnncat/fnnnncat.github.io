安装idea并启动maven项目

手上需要写一个后端为maven工程的项目，没有测试环境，为了方便只能启本地的后台端口，于是准备装下idea，启动下后台进行联调

maven 梗概

首先，maven是什么呢？在我看来和前端的包管理概念类似，maven在java中相当于npm在js中的地位，其中的pom.xml与package.json作用类似，但pom.xml是扁平化的管理，不是如npm一样的嵌套式管理（现在的yarn也是扁平化的管理）但是java和js的构建有着很大的区别,maven出现的要比npm早，也大的多，定制化的插件庞大，又是maven会用来管理非Java项目。

idea 安装

 idea安装包下载，注册，启动 （create associations 是选择后带有选项后缀的文件，都将使用该软件打开）

jdk 安装

正常安装

导入maven项目

maven的仓库会分为本地仓库和远程仓库，远程仓库会进一步分为中央仓库、私服和其他仓库，因为在云桌面内网开发，所以配置了本地仓库和私服。

  1. 配置setting.xml文件。 默认maven会创建，一个.m2的文件夹，其中含有setting.xml

    • 默认jdk使用java8
    • 配置私服和本地仓库地址
    <settings>
  <localRepository>D:\64\m2\repository</localRepository>
</settings>

  2. 创建repository文件夹，根据本地仓库地址创建文件夹

  3. import 项目文件夹，选择jdk，选择项目右键‘maven–>reimport’

  4. file–>build–>maven 配置maven/setting/repository的地址

  5. 配置run configure: application –> main class–>vmoptions 加上当前运行参数

  6. 结束，正常启动
